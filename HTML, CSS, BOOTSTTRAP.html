Basics & Document Structure
Q. What is HTML? Differences between HTML & HTML5? Advantages of HTML5?
▾

HTML (HyperText Markup Language) is the standard language used to create and structure content on the web. It uses tags (like <p>, <div>, <h1>) to tell the browser how to display text, images, links, and other content. It is NOT a programming language — it cannot do logic. It just describes structure.

HTML (4)HTML5
Requires Flash for audio/videoNative <audio> & <video> tags
No semantic tags (only <div>)Semantic tags: <header>, <nav>, <section>, <footer>
No localStorage or sessionStorageFull Web Storage API
No canvas drawing<canvas> tag for 2D/3D drawing
Loose doctype declarationSimple: <!DOCTYPE html>
No geolocation supportBuilt-in Geolocation API

Advantages of HTML5

  • Built-in multimedia support (no plugins needed)
  • Semantic tags improve readability and SEO
  • Offline storage via localStorage/sessionStorage
  • Better form controls (date, range, email, color pickers)
  • Canvas & SVG for graphics without Flash
Q. What is the difference between HTML and XHTML?
▾

XHTML (Extensible HyperText Markup Language) is a stricter, XML-based version of HTML. It enforces rules that HTML is lenient about.

HTMLXHTML
Tags can be lowercase or UPPERCASEAll tags must be lowercase
Tags don't need to be closed: <br>All tags must be self-closed: <br />
Attribute values don't need quotesAll attribute values must be quoted
Browser forgives errorsOne error breaks the whole page
Loose parsingStrict XML parsing
💡 In modern development, HTML5 has replaced XHTML. You rarely write XHTML today, but interviewers ask about it to test your understanding of standards.
Q. What is the role of DOCTYPE in HTML?
▾

DOCTYPE (Document Type Declaration) is the very first line in an HTML file. It tells the browser which version of HTML you're using so it can render the page correctly. Without it, the browser may enter "quirks mode" — an old compatibility mode where CSS and layout behave unpredictably.

<!-- HTML5 DOCTYPE (modern, simple) -->
<!DOCTYPE html>
<html lang="en">
  <head>...
  <body>...
  • Not an HTML tag — it's an instruction to the browser
  • Must be the very first thing in the file (before <html>)
  • HTML5 DOCTYPE is simply <!DOCTYPE html> — no version number needed
  • Older HTML4 had long complex DOCTYPE strings
Q. What if you remove DOCTYPE html from your HTML?
▾

If you remove DOCTYPE, the browser switches to Quirks Mode — a backward-compatibility mode that mimics old browser behaviour from the 1990s. This causes:

  • CSS box model behaves differently (widths include padding in quirks mode)
  • Layout inconsistencies across browsers (Chrome, Firefox, IE all differ)
  • Fonts and line heights render differently
  • Some HTML5 features may not work correctly
✅ Always include <!DOCTYPE html> as the first line. It's a one-liner that saves you from hours of debugging.

Where to place JavaScript?

<!-- ✅ Best practice: Just before </body> -->
<body>
  <h1>Hello</h1>
  <script src="app.js"></script>
</body>

<!-- OR in <head> with defer attribute -->
<head>

💡 Placing JS before </body> ensures the HTML loads first, then JS runs. If you place it in <head> without defer, JS blocks page rendering.
Q. What is the Title Tag in HTML? What are the 4 advantages of the Title tag?
▾

The <title> tag defines the title of the web page. It appears in the browser tab and is used by search engines. It lives inside <head>.

<head>
  <title>My Portfolio - Home</title>
</head>

4 Advantages

  • Browser Tab: Shows the page name so users can identify the tab
  • SEO: Search engines (Google) use the title as the clickable headline in search results
  • Bookmarks: When you bookmark a page, the title becomes the bookmark name
  • Accessibility: Screen readers announce the title first, helping visually impaired users
Q. What are Meta Tags? What are the 5 types of meta tags?
▾

Meta tags live inside <head> and provide metadata (information about the page) to browsers, search engines, and social media platforms. They are self-closing and invisible to users.

<!-- 1. Charset - defines character encoding -->
<meta charset="UTF-8">

<!-- 2. Viewport - makes site responsive on mobile -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">

<!-- 3. Description - shown in Google search results -->
<meta name="description" content="Best portfolio site for web developers">

<!-- 4. Keywords - for SEO (less important now) -->
<meta name="keywords" content="HTML, CSS, JavaScript, React">

<!-- 5. Author -->
<meta name="author" content="Rahul Sharma">

▾

An HTML Element is the complete unit: opening tag + content + closing tag. A Tag is just the marker itself (the angle-bracket part).

<!-- TAG = just <p> or </p> -->
<!-- ELEMENT = the whole thing below -->

  • Some elements are self-closing (void elements): <br>, <img>, <input>, <hr>

  • Every tag has a purpose — never use a tag just for its visual appearance
Q. What are the roles and uses of the <div> element in HTML?
▾

A <div> (division) is a generic block-level container with no semantic meaning. It's used purely for layout, grouping, and styling purposes.

</div>

<!-- Used as flex/grid containers -->
<div class="container">
  <div class="col">Left</div>
  <div class="col">Right</div>
</div>
✅ Use semantic tags (<section>, <article>, <nav>) when the content has meaning. Use <div> only when you just need a wrapper with no semantic meaning.

▾

  • Proper heading hierarchy helps Google index your content correctly
  • Don't skip levels (don't go h1 → h4) — Google notices
  • Screen readers use headings for navigation — also improves accessibility

HTML Elements & Tags – Part II
Q. What are Root, Parent, Child & Nested elements?
▾

HTML elements have a family tree structure (called the DOM tree):

<html>           <!-- ROOT element -->
  <body>         <!-- PARENT of div -->
    <div>        <!-- CHILD of body, PARENT of p -->
      <p>        <!-- CHILD of div (NESTED element) -->
        <span>   <!-- NESTED inside p (deeply nested) -->
        </span>
      </p>
    </div>
  </body>
</html>
  • Root: The top-most element — always <html>
  • Parent: Any element that contains another element inside it
  • Child: An element directly inside a parent element
  • Nested: Elements placed inside other elements (any depth)
Q. What are Empty Elements?
▾

Empty elements (also called void elements) are HTML tags that have NO closing tag and NO content between tags. They are self-contained.

<!-- These are ALL empty/void elements -->
<br>       <!-- line break -->
<hr>       <!-- horizontal rule -->
<img src="photo.jpg" alt="photo">
<input type="text">
<link rel="stylesheet" href="style.css">
<meta charset="UTF-8">
💡 In XHTML you'd write <br /> with a self-closing slash. In HTML5, both <br> and <br /> are valid.
Q. What are Semantic Elements in HTML? Is div a semantic element?
▾

Semantic elements are tags that carry meaning about the content they contain. They tell the browser AND the developer what the content IS, not just how to display it.

Non-Semantic (no meaning)Semantic (has meaning)
<div><header>
<span><nav>
<main>
<section>
<article>
<aside>
<footer>
💡 Is <div> semantic? NO. <div> means nothing — it's just a box. Always prefer semantic elements when you know the role of the content.
Q. Can HTML tags be written in Uppercase?
▾

In HTML5: Yes, both uppercase and lowercase tags are valid. The browser accepts both. In XHTML: No — all tags must be lowercase.

<!-- Both work in HTML5 -->
<P>This works</P>
<p>This also works</p>
✅ Best practice: Always write HTML tags in lowercase. It's the modern standard, more readable, and compatible with XHTML.

HTML Attributes
Q. What are HTML Attributes? What are the Types of HTML attributes?
▾

Attributes provide additional information about an HTML element. They are always written in the opening tag as name="value" pairs.

<!-- General syntax -->
<tagname attribute="value">content</tagname>

<!-- Real examples -->
<img src="photo.jpg" alt="Profile photo" width="200">
<a href="https://google.com" target="_blank">Link</a>
<input type="email" placeholder="Enter email" required>

Types of Attributes

  • Global attributes: Work on any element — id, class, style, title, hidden, data-*
  • Element-specific: src (img), href (a), action (form), for (label)
  • Boolean attributes: Just their presence means true — disabled, required, checked, readonly
  • Event attributes: onclick, onchange, onsubmit, onload
Q. What are the Id, Style & Class attributes? When to use what?
▾
AttributePurposeWhen to Use
idUnique identifier for ONE elementWhen you need to target a specific single element in CSS or JS
classGroup identifier for MULTIPLE elementsWhen multiple elements share the same style
styleInline CSS directly on the elementFor quick one-off overrides (avoid overusing)
<!-- id: unique, used once -->
<div id="main-header">Only one header</div>

<!-- class: reusable across many elements -->
<button class="btn-primary">Submit</button>
<button class="btn-primary">Cancel</button>

<!-- style: inline, use sparingly -->
<p style="color: red; font-size: 18px;">Error message</p>
Q. What will happen if two elements have the same ids?
▾

It is a violation of HTML rules. IDs must be unique per page. However, browsers are forgiving — they won't throw an error. But the consequences are serious:

<!-- BAD - duplicate IDs -->
<div id="box">First</div>
<div id="box">Second</div>

// In JavaScript - getElementById only returns FIRST match
document.getElementById('box'); // only gets "First"

// querySelectorAll('#box') may behave inconsistently
  • CSS will style only the first element with that ID
  • JavaScript getElementById only finds the first one
  • Accessibility tools get confused
  • HTML validation will fail
Q. How to specify Multiple Classes for a single element? What is the style precedence?
▾

Separate multiple class names with a space inside the class attribute:

<!-- Multiple classes on one element -->
<div class="card featured large shadow">
  This div has 4 classes applied
</div>

Style Precedence (Specificity)

  • Highest: !important always wins
  • 2nd: Inline style (style="...")
  • 3rd: ID selector (#myId)
  • 4th: Class selector (.myClass)
  • 5th (lowest): Element selector (p, div, h1)
💡 When two rules have equal specificity, the one that appears LAST in the CSS file wins.
Q. What are Data Attributes in HTML?
▾

Data attributes let you store custom data directly on HTML elements. They start with data- and you can name them anything.

<!-- Storing custom data on elements -->
<button data-user-id="42" data-action="delete">
  Delete User
</button>

<div data-product-id="P001" data-price="999">iPhone</div>

// Access in JavaScript via dataset
const btn = document.querySelector('button');
console.log(btn.dataset.userId);  // "42"
console.log(btn.dataset.action);  // "delete"
✅ Very useful in real projects! Instead of making extra API calls or storing data in JavaScript variables, you can embed data right in the HTML and read it with JS.

Q. What are HTML Entities? What are the Top 5 Emerging Trends in HTML?
▾

HTML Entities are special codes used to display reserved characters (like < > &) that HTML would otherwise interpret as code.

<!-- Without entity, < breaks HTML -->

  • AI-generated HTML: Tools like GitHub Copilot and Claude writing HTML/CSS

CSS Basics & Implementation
Q. What is CSS? What are the 5 ways to Implement CSS in HTML?
▾

CSS (Cascading Style Sheets) is the language used to style HTML elements — controlling colours, fonts, layouts, spacing, animations, and responsive behaviour. HTML builds the structure; CSS makes it look good.

/* 1. Inline Style - directly on element */
<p style="color: blue; font-size: 18px;">Hello</p>

/* 2. Internal / Embedded - inside <style> in <head> */
<style>
  p { color: blue; }
</style>

/* 3. External stylesheet - separate .css file */
<link rel="stylesheet" href="style.css">

/* 4. @import inside CSS file */
@import url('buttons.css');

/* 5. JavaScript (dynamically adding styles) */
document.getElementById('box').style.color = 'red';
Q. Inline vs Internal vs External Stylesheet — when to use each? What are the 5 advantages of External?
▾
TypeWhen to UseDrawback
InlineQuick one-off override, email templates, JS dynamic stylingNot reusable, hard to maintain
InternalSingle-page apps, email HTML, quick prototypesStyles don't apply to other pages
ExternalAll real projects — any multi-page siteExtra HTTP request (tiny, negligible)

5 Advantages of External Stylesheet

  • Reusability: One CSS file styles all pages — change once, updates everywhere
  • Separation of concerns: HTML for structure, CSS for style — cleaner code
  • Caching: Browser caches the CSS file, so subsequent pages load faster
  • Maintainability: Much easier to find and edit styles in a dedicated file
  • Team collaboration: Developers and designers can work on HTML and CSS independently
Q. What is CSS Preprocessors? What is SCSS? What are the 3 Types?
▾

A CSS Preprocessor is a tool that extends CSS with programming features (variables, nesting, mixins, functions) and compiles down to regular CSS that browsers understand.

/* Regular CSS - repetitive */
.btn-primary { background: #3b82f6; color: white; padding: 10px 20px; }
.btn-danger  { background: #ef4444; color: white; padding: 10px 20px; }

/* SCSS - uses variables, nesting, mixins */
$primary: #3b82f6;
$padding: 10px 20px;

@mixin btn($bg) {
  background: $bg;
  color: white;
  padding: $padding;
}

.btn-primary { @include btn($primary); }
.btn-danger  { @include btn(#ef4444); }

/* Nesting in SCSS */
.navbar {
  background: #333;
  a {                     /* compiles to .navbar a */
    color: white;
    &:hover { color: #3b82f6; }
  }
}

3 Types of CSS Preprocessors

  • SASS/SCSS: Most popular. SCSS syntax is identical to CSS with extra features. Most React/Next.js projects use this.
  • LESS: Similar to SCSS but uses JavaScript for compilation. Used in older Bootstrap (v3).
  • Stylus: Flexible, minimalist syntax — less popular but very powerful.
Selectors in CSS
Q. What are Selectors in CSS? How many types? ID vs Element vs Class? Universal? Descendant? Attribute? Child? Pseudo-class & Pseudo-element?
▾

A CSS Selector is the pattern that tells CSS which HTML element(s) to style.

/* 1. Element selector - targets all <p> tags */
p { color: gray; }

/* 2. Class selector - targets all .btn elements */
.btn { padding: 10px; }

/* 3. ID selector - targets ONE #header element */
#header { background: #333; }

/* 4. Universal selector - targets EVERY element */
* { box-sizing: border-box; margin: 0; }

/* 5. Descendant selector - any p inside .container (any depth) */
.container p { color: blue; }

/* 6. Child selector (>) - only DIRECT children */
.container > p { color: red; }

/* 7. Attribute selectors */
input[type="email"] { border: 1px solid blue; }  /* exact match */
a[href^="https"] { color: green; }           /* starts with */
a[href$=".pdf"] { color: red; }              /* ends with */
a[href*="google"] { font-weight: bold; }     /* contains */

/* 8. Pseudo-class - element in a specific STATE */
a:hover { color: orange; }          /* when mouse hovers */
input:focus { outline: 2px solid blue; }
li:first-child { font-weight: bold; }
li:nth-child(2) { color: red; }

/* 9. Pseudo-element - styles a PART of the element */
p::first-line { font-weight: bold; }
p::first-letter { font-size: 2em; }
.btn::before { content: '→ '; }     /* adds content before */
.btn::after  { content: ' ✓'; }     /* adds content after */
SelectorSpecificityUse When
Element (p)Lowest (0,0,1)Global base styles
Class (.btn)Medium (0,1,0)Reusable component styles
ID (#header)High (1,0,0)Unique one-off elements
💡 Child (>) vs Descendant (space): Child selector only targets DIRECT children. Descendant targets children at ANY depth. Example: .nav > a targets only anchors directly inside .nav, while .nav a targets anchors anywhere inside .nav (including inside nested divs).
Layout and Positioning
Q. Explain the CSS Box Model.
▾

Every HTML element is a rectangular box. The Box Model describes the 4 layers around that content:

/* Box Model layers from inside out: */
/*  Content → Padding → Border → Margin */

.box {
  width: 200px;         /* content area */
  padding: 20px;        /* space INSIDE the border */
  border: 5px solid #333; /* the border line */
  margin: 30px;         /* space OUTSIDE the border */
}
/* Default box-sizing: content-box */
/* Total width = 200 + 20+20 + 5+5 = 250px */

/* Better: border-box includes padding+border in width */
* { box-sizing: border-box; }
/* Now: total width stays 200px (padding/border included) */
✅ Always use * { box-sizing: border-box; } in every project. It makes width calculations predictable and eliminates layout surprises.
Q. What is the difference between Padding and Margin?
▾
PaddingMargin

Has the element's background colourAlways transparent

Cannot be negativeCan be negative (used in special layouts)

.card {
  padding: 20px;        /* space inside — feels spacious inside */
  margin: 30px auto;    /* centers horizontally, 30px top/bottom gap */
  margin-top: -10px;   /* negative margin = pulls element up */
}

Q. What is z-index in CSS? How are same z-index elements displayed? What is the float property?
▾

z-index controls the stacking order of elements along the Z-axis (towards/away from the viewer). Higher z-index = appears in front. Only works on positioned elements (relative, absolute, fixed, sticky).

.modal-backdrop {
  position: fixed;
  z-index: 100;    /* behind modal */
}
.modal {
  position: fixed;
  z-index: 200;    /* on top of backdrop */
}
.tooltip {
  position: absolute;
  z-index: 999;   /* on top of everything */
}
💡 If two elements have the SAME z-index, the one that appears LATER in the HTML is shown on top (normal document flow order).

float property

/* float: was the old way to do layouts (before Flexbox) */
img {
  float: left;    /* text wraps around image on the right */
  margin: 0 15px 10px 0;
}
/* clearfix: needed to fix parent collapse after float */
.clearfix::after {
  content: '';
  display: block;
  clear: both;
}
✅ In modern development, float is only used for wrapping text around images. Use Flexbox or Grid for all layouts.

Responsive Design, Flexbox & Grid
Q. What is the meta viewport tag? What are Media Queries?
▾
<!-- viewport meta tag: makes layout scale correctly on mobile -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<!-- Without this, mobile browsers zoom out to show desktop layout -->

Media Queries apply different CSS rules based on screen size, device type, or other conditions. They are the foundation of responsive design.

/* Mobile-first approach (preferred) */
/* Base styles for mobile */
.container { padding: 16px; }

/* Tablet: 768px and up */
@media (min-width: 768px) {
  .container { padding: 32px; }
}

/* Desktop: 1024px and up */
@media (min-width: 1024px) {
  .container { max-width: 1200px; margin: 0 auto; }
}

/* Multiple conditions */
@media (min-width: 768px) and (max-width: 1024px) {
  /* tablets only */
}

/* Dark mode */
@media (prefers-color-scheme: dark) {
  body { background: #111; color: white; }
}
Q. What is display: flex in CSS? (Flexbox)
▾

Flexbox is a 1-dimensional layout system (works in a row OR column). It's perfect for aligning items in a navbar, centering content, or distributing space between cards.

/* PARENT (flex container) */
.navbar {
  display: flex;
  flex-direction: row;          /* row (default) | column */
  justify-content: space-between; /* main axis alignment */
  /* flex-start | center | flex-end | space-around | space-evenly */
  align-items: center;          /* cross axis alignment */
  /* flex-start | flex-end | stretch | baseline */
  flex-wrap: wrap;              /* wraps items to next line */
  gap: 16px;                    /* space between items */
}

/* CHILD (flex item) */
.logo { flex: 0 0 auto; }      /* don't grow, don't shrink */
.nav-links { flex: 1; }          /* take all remaining space */

/* Perfect centering with flexbox */
.centered {
  display: flex;
  justify-content: center;
  align-items: center;
  min-height: 100vh;
}
Q. What is CSS Grid Layout?
▾

CSS Grid is a 2-dimensional layout system — it controls both rows AND columns simultaneously. Perfect for full page layouts, dashboards, and card grids.

/* Basic grid */
.grid {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr;  /* 3 equal columns */
  grid-template-columns: repeat(3, 1fr); /* same as above */
  gap: 24px;
}

/* Responsive grid without media queries */
.auto-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
  /* auto-fit: creates as many columns as fit, minimum 250px each */
  gap: 20px;
}

/* Full page layout */
.page {
  display: grid;
  grid-template-areas:
    "header header"
    "sidebar main"
    "footer footer";
  grid-template-columns: 250px 1fr;
  grid-template-rows: auto 1fr auto;
  min-height: 100vh;
}
header  { grid-area: header; }
aside   { grid-area: sidebar; }
main    { grid-area: main; }
footer  { grid-area: footer; }
💡 Flexbox vs Grid: Use Flexbox for 1D layouts (nav, buttons in a row, centering). Use Grid for 2D layouts (full pages, card grids with aligned rows AND columns).
Q. How can you create a Sticky Header in CSS? What are 5 best practices for Responsive Design?
▾
/* Sticky header - sticks to top when you scroll past it */
header {
  position: sticky;
  top: 0;                  /* sticks at 0px from top */
  z-index: 100;            /* stays above page content */
  background: white;
  box-shadow: 0 2px 8px rgba(0,0,0,0.1);
}

/* For fixed header (always stays, even before scroll) */
header {
  position: fixed;
  top: 0; left: 0; right: 0;
  z-index: 100;
}
body { padding-top: 70px; }  /* compensate for header height */

5 Best Practices for Responsive Design

  • Mobile-first: Write CSS for small screens first, then use min-width media queries to scale up
  • Use relative units: %, em, rem, vw, vh instead of fixed px for fonts and spacing
  • Flexible images: Always add img { max-width: 100%; height: auto; }
  • Use Flexbox/Grid: Avoid floats and fixed widths — Grid/Flex adapt naturally
  • Test on real devices: Use Chrome DevTools device emulator at every breakpoint

Bootstrap Fundamentals
Q. What is Bootstrap? What are the other 5 responsive design frameworks?
▾

Bootstrap is a free, open-source CSS + JavaScript framework created by Twitter. It gives you a ready-made library of responsive layout classes, pre-designed UI components (navbars, modals, cards), and utility classes — so you can build professional sites without writing CSS from scratch.

  • Tailwind CSS: Utility-first, highly customizable, very popular in 2024
  • Foundation: By Zurb — more flexible than Bootstrap, used in enterprise
  • Bulma: Pure CSS, no JavaScript, clean and modern
  • Materialize CSS: Based on Google's Material Design
  • Semantic UI: Human-friendly class names (class="ui button" etc.)
Q. What are the 5 Advantages of using Bootstrap?
▾
  • Speed: Pre-built components mean you build pages much faster than writing CSS from scratch
  • Responsive by default: Its grid system automatically adapts to all screen sizes — mobile, tablet, desktop
  • Cross-browser compatible: Bootstrap handles browser inconsistencies for you
  • Huge community: Massive documentation, thousands of themes and templates available
  • Consistent design: All components follow the same visual language — no mismatched styles across pages

💡 CDN is great for quick projects. NPM is better for React/Vue apps where you want to import only what you need and customize with SCSS variables.
Q. Explain the Grid System in Bootstrap?
▾

Bootstrap's grid divides every row into 12 columns. You assign each element how many of those columns it should take on different screen sizes. Everything is wrapped in .container → .row → .col-*.

<!-- Basic Grid -->
<div class="container">
  <div class="row">

    <!-- Takes 6 of 12 columns = 50% width -->
    <div class="col-6">Left Half</div>
    <div class="col-6">Right Half</div>

  </div>
  <div class="row">

    <!-- Responsive: full width on mobile, 3 cols on desktop -->
    <div class="col-12 col-md-4">Card 1</div>
    <div class="col-12 col-md-4">Card 2</div>
    <div class="col-12 col-md-4">Card 3</div>

  </div>
</div>
✅ The 12-column system lets you do: full width (12), halves (6+6), thirds (4+4+4), quarters (3+3+3+3), sidebar layout (3+9), etc.
