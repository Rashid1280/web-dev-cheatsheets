Q.01 How to execute a piece of code repeatedly after some fixed time? ▼

Use setInterval(callback, delay) to run a function repeatedly at a fixed interval. It returns an ID so you can stop it later with clearInterval(id). For async or heavy tasks, prefer a recursive setTimeout pattern — it waits for the previous execution to finish before scheduling the next, preventing pile-up.

javascript
// setInterval — fires every 1 second
const id = setInterval(() => {
  console.log("Tick!");
}, 1000);

// Stop after 5 ticks
setTimeout(() => clearInterval(id), 5000);

// Countdown timer example
let remaining = 10;
const timer = setInterval(() => {
  console.log(remaining--);
  if (remaining < 0) clearInterval(timer);
}, 1000);

// Recursive setTimeout — safer for async tasks
function poll() {
  fetch("/api/status")
    .then(r => r.json())
    .then(data => {
      process(data);
      setTimeout(poll, 3000); // only schedules AFTER current completes
    });
}
poll();
Q.02 How to handle asynchronous operations in JS? ▼

The modern standard is async/await with try/catch. It makes async code read like synchronous code, handles errors naturally, and works well with conditionals and loops. Under the hood it's still Promises — await unwraps a Promise and pauses the function without blocking the main thread.

javascript
// async/await — most readable
async function loadUser(id) {
  try {
    const res  = await fetch(`/api/users/${id}`);
    if (!res.ok) throw new Error(`HTTP ${res.status}`);
    const user = await res.json();
    render(user);
  } catch (err) {
    showError(err.message);
  } finally {
    hideSpinner(); // always runs
  }
}

// Sequential — each step uses result of previous
const user    = await fetchUser(id);
const profile = await fetchProfile(user.profileId);

// Parallel — run independently at same time
const [users, posts] = await Promise.all([
  fetch("/api/users").then(r => r.json()),
  fetch("/api/posts").then(r => r.json()),
]);
Q.03 How to manipulate and modify CSS styles of HTML elements dynamically? ▼

Two approaches: element.style for direct inline changes, and classList to add/remove/toggle CSS classes. The classList approach is preferred in real applications — it keeps styling logic in CSS files and is easier to manage. CSS variables can also be updated via JS for theme-wide changes.

javascript
const el = document.querySelector(".card");

// Inline style — camelCase property names
el.style.backgroundColor = "#1a1a1a";
el.style.fontSize        = "18px";
el.style.display         = "none";  // hide
el.style.display         = "";      // revert to CSS value

// classList — preferred
el.classList.add("active");
el.classList.remove("hidden");
el.classList.toggle("dark");         // on/off switch
el.classList.toggle("open", isOpen); // force based on boolean
el.classList.contains("active");    // true/false
el.classList.replace("old", "new"); // swap classes

// Update CSS variable for theme-wide change
document.documentElement.style
  .setProperty("--primary", "#ff6600");
Q.04 How to handle errors and exceptions in your code? ▼

Use try/catch/finally blocks. Always throw Error objects rather than strings — they carry a stack trace. Use custom error classes extending Error for domain-specific errors. Never silently swallow errors with an empty catch block.

javascript
// try/catch/finally
try {
  const data = JSON.parse(rawInput);
  processData(data);
} catch (err) {
  if (err instanceof SyntaxError) {
    showError("Invalid JSON");
  } else {
    throw err; // re-throw unknown errors
  }
} finally {
  hideSpinner(); // always runs
}

// Custom error class
class ValidationError extends Error {
  constructor(msg, field) {
    super(msg);
    this.name  = "ValidationError";
    this.field = field;
  }
}
throw new ValidationError("Required", "email");

// Async error handling
async function save(data) {
  try {
    await uploadToServer(data);
  } catch (err) {
    console.error("Save failed:", err);
  }
}
Q.05 How to store key-value pairs and efficiently access and manipulate the data? ▼

Use a plain object for static string keys, or a Map when keys are dynamic, non-string, or when you need guaranteed insertion order and a built-in .size. Maps are also faster for frequent additions and deletions.

javascript
// Plain object — string keys, known structure
const user = { name: "Arun", age: 25 };
user.email    = "a@b.com"; // add
user.age      = 26;       // modify
delete user.age;           // remove
"name" in user;           // true — check existence

// Map — any key type, ordered, with .size
const cache = new Map();
cache.set("user:1", { name: "Arun" });
cache.set(42, "numeric key!");
cache.get("user:1");  // { name: "Arun" }
cache.has(42);        // true
cache.delete(42);
cache.size;            // 1

// Iterate Map
for (const [key, val] of cache) {
  console.log(key, val);
}
Q.06 How to iterate over elements in an array and perform a specific operation on each element? ▼

Pick based on what you need back: forEach for side effects only, map to transform into a new array, filter for a subset, reduce to accumulate a value, or for...of when you need break/continue or async/await inside the loop.

javascript
const nums = [1, 2, 3, 4, 5];

// forEach — side effects, returns undefined
nums.forEach((n, i) => console.log(i, n));

// map — transform, returns new array
const doubled = nums.map(n => n * 2); // [2,4,6,8,10]

// filter — subset, returns new array
const evens = nums.filter(n => n % 2 === 0); // [2,4]

// reduce — accumulate to single value
const sum = nums.reduce((acc, n) => acc + n, 0); // 15

// for...of — supports break, continue, await
for (const n of nums) {
  if (n === 3) break;
  console.log(n); // 1, 2
}

Create elements with document.createElement(), configure them, then insert with appendChild / prepend / insertAdjacentElement. Remove with element.remove(). For inserting multiple items efficiently, use a DocumentFragment to batch DOM updates into one reflow.

javascript
// ADD — create → configure → insert
const card = document.createElement("div");
card.className   = "card";
card.textContent = "New Card";
card.setAttribute("data-id", "42");
document.getElementById("container").appendChild(card);

// insertAdjacentHTML — quick HTML string insertion
document.querySelector("#anchor")
  .insertAdjacentHTML("afterend", "<p>Inserted!</p>");

// REMOVE
card.remove();                       // modern
card.parentNode.removeChild(card);  // older compatible way

// DocumentFragment — batch insert (1 reflow instead of N)
const frag = document.createDocumentFragment();
["A", "B", "C"].forEach(text => {
  const li = document.createElement("li");
  li.textContent = text;
  frag.appendChild(li);
});
document.querySelector("ul").appendChild(frag);

Q.09 How to manage the state in a web application? ▼

State management scales with your app. Use local variables or closures for component-level state, localStorage / sessionStorage for persistent client state, or a simple store pattern (observer/pub-sub) for shared state across multiple parts of the UI without a framework.

javascript
// Simple store with observer pattern
function createStore(initialState) {
  let state     = { ...initialState };
  const listeners = [];
  return {
    getState()     { return { ...state }; },
    setState(next) {
      state = { ...state, ...next };
      listeners.forEach(fn => fn(state)); // notify all subscribers
    },
    subscribe(fn)  { listeners.push(fn); }
  };
}

const store = createStore({ count: 0, user: null });
store.subscribe(state => renderUI(state));
store.setState({ count: 1 }); // triggers renderUI automatically

// Persist state to localStorage
localStorage.setItem("appState", JSON.stringify(store.getState()));
// Restore on page load
const saved = JSON.parse(localStorage.getItem("appState")) || {};
Q.10 How to implement a queue or a stack-like data structure in JS? ▼

Both can be built with JavaScript arrays. A Stack is LIFO (Last In, First Out) — uses push to add and pop to remove from the end. A Queue is FIFO (First In, First Out) — uses push to enqueue and shift to dequeue from the front.

javascript
// STACK — LIFO (undo/redo, browser history)
class Stack {
  constructor() { this.items = []; }
  push(item)  { this.items.push(item); }
  pop()       { return this.items.pop(); }
  peek()      { return this.items.at(-1); }
  isEmpty()   { return this.items.length === 0; }
  get size()  { return this.items.length; }
}
const stack = new Stack();
stack.push(1); stack.push(2); stack.push(3);
stack.pop(); // 3 — last in, first out

// QUEUE — FIFO (task queue, print queue)
class Queue {
  constructor() { this.items = []; }
  enqueue(item) { this.items.push(item); }
  dequeue()     { return this.items.shift(); }
  front()       { return this.items[0]; }
  isEmpty()     { return this.items.length === 0; }
}
const q = new Queue();
q.enqueue("A"); q.enqueue("B"); q.enqueue("C");
q.dequeue(); // "A" — first in, first out
Q.11 How do you attach an event handler to an HTML element? ▼

Use addEventListener(event, handler) — the modern standard. It supports multiple handlers on the same element, and provides options like once, passive, and capture. Named functions can be removed later with removeEventListener. Use event delegation on a parent element to handle many children efficiently.

javascript
const btn = document.querySelector("#myBtn");

// Named function — can be removed later
function handleClick(e) {
  console.log("Clicked:", e.target);
}
btn.addEventListener("click", handleClick);
btn.removeEventListener("click", handleClick); // ✅ removable

// Options object
btn.addEventListener("click", handleClick, {
  once:    true,  // auto-removes after first fire
  passive: true,  // hints browser won't call preventDefault
  capture: false  // bubble phase (default)
});

// Event delegation — one listener on parent handles all children
document.getElementById("list")
  .addEventListener("click", (e) => {
    if (e.target.matches("li")) handleItem(e.target);
  });

Q.13 How to fetch data from multiple APIs in parallel and process the results together? ▼

Use Promise.all() to fire all requests simultaneously and wait for all to succeed. Total time equals the slowest single request — much faster than sequential awaits. Use Promise.allSettled() when some requests may fail but you still want the successful results.

javascript
// Promise.all — all must succeed
async function loadDashboard() {
  const [users, products, orders] = await Promise.all([
    fetch("/api/users").then(r => r.json()),
    fetch("/api/products").then(r => r.json()),
    fetch("/api/orders").then(r => r.json()),
  ]); // Total time ≈ slowest single request ✅
  renderDashboard({ users, products, orders });
}

// Promise.allSettled — continue even if some fail
async function loadAll(ids) {
  const results = await Promise.allSettled(
    ids.map(id => fetch(`/api/item/${id}`).then(r => r.json()))
  );
  results.forEach((r, i) => {
    if (r.status === "fulfilled") render(r.value);
    else console.warn(`id ${ids[i]} failed:`, r.reason);
  });
}
Q.14 What are the methods to manipulate JSON data efficiently? ▼

JSON.parse() converts a JSON string into a JS object so you can work with it. JSON.stringify() converts a JS object back into a JSON string for sending to APIs or storage. Once parsed, use standard array and object methods to transform the data.

javascript
// Parse — JSON string → JS object
const obj = JSON.parse('{"name":"Arun","skills":["JS","React"]}');
obj.name;        // "Arun"
obj.skills[0];   // "JS"

// Stringify — JS object → JSON string
JSON.stringify({ name: "Arun" });           // '{"name":"Arun"}'
JSON.stringify({ name: "Arun" }, null, 2); // pretty-printed

// Deep clone via JSON (loses functions, Dates, undefined)
const clone = JSON.parse(JSON.stringify(obj));

// Transform an array of JSON objects
const users  = JSON.parse(apiResponse);
const names  = users.map(u => u.name);
const active = users.filter(u => u.isActive);
const byId   = Object.fromEntries(users.map(u => [u.id, u]));

// Safe parse with fallback
function safeJSON(str, fallback = {}) {
  try { return JSON.parse(str); }
  catch { return fallback; }
}
Q.15 How to get the current URL of a webpage? ▼

Use window.location — it gives you the full URL and all its parts as separate properties. Use the URL constructor or URLSearchParams to parse query parameters cleanly without string manipulation.

javascript
// Full URL and its parts
window.location.href;      // "https://example.com/page?q=js#top"
window.location.origin;    // "https://example.com"
window.location.pathname;  // "/page"

window.location.hash;      // "#top"

// Parse query params — modern way

const url = new URL("https://example.com/path?id=99");
url.searchParams.get("id"); // "99"

// Navigate programmatically
window.location.href = "/new-page";      // redirect (adds history)
window.location.replace("/new-page");   // redirect (no history)
window.history.pushState({}, "", "/new-path"); // SPA navigation
Q.16 How do you find the length of an array in JS? ▼

Use the .length property — it's automatically maintained and always reflects the current number of elements. It can also be set to truncate or clear an array. For objects, use Object.keys(obj).length since objects don't have a length property.

javascript
const fruits = ["apple", "banana", "cherry"];
fruits.length;            // 3

// Last element via length
fruits[fruits.length - 1]; // "cherry" (classic)
fruits.at(-1);            // "cherry" (modern)

// Check if empty
fruits.length === 0;      // false

// Truncate by setting length
fruits.length = 2;
console.log(fruits);      // ["apple", "banana"] — cherry gone

// Clear entire array
fruits.length = 0;        // [] — all gone

// Object — no .length, use Object.keys
const obj = { a: 1, b: 2, c: 3 };
Object.keys(obj).length;  // 3
Q.17 How to create a copy of an array? ▼

For primitive-only arrays, a shallow copy with spread [...arr] or arr.slice() is sufficient. If the array contains objects or nested arrays, use structuredClone() for a fully independent deep copy — otherwise changes to nested objects will still affect the original.

javascript
const original = [1, 2, 3];

// Shallow copy — all equivalent
const c1 = [...original];          // spread — most common
const c2 = original.slice();      // slice with no args
const c3 = Array.from(original);  // from

c1.push(99);
console.log(original); // [1, 2, 3] — unaffected ✅

// Problem with shallow copy — nested objects share references
const arr    = [{ x: 1 }, { x: 2 }];
const shallow = [...arr];
shallow[0].x = 99;
console.log(arr[0].x);   // 99 — original mutated! ⚠️

// Deep copy — fully independent
const deep = structuredClone(arr);
deep[0].x = 999;
console.log(arr[0].x);   // 99 — original unaffected ✅
Q.18 How do you access individual characters in a string? ▼

Access by index using bracket notation str[i] or charAt(i). The modern at() method also supports negative indices. Remember strings are immutable — you can read characters but not change them in place.

javascript
const str = "JavaScript";

str[0];           // "J" — bracket notation (read-only)
str.charAt(4);    // "S" — same result
str.at(0);        // "J" — modern
str.at(-1);       // "t" — last character
str.at(-6);       // "S" — 6th from end

// Strings are immutable — assignment silently fails
str[0] = "j";
console.log(str); // still "JavaScript"

// Iterate every character
for (const char of str) console.log(char);
[...str];          // ["J","a","v","a","S","c","r","i","p","t"]

// Unicode code points
str.charCodeAt(0);     // 74 — numeric code of "J"
String.fromCharCode(74); // "J"
Q.19 How can you check if a string contains a specific substring? ▼

The modern and cleanest way is string.includes() — returns true or false. Use startsWith / endsWith for positional checks, indexOf when you also need the position, and a RegExp for pattern matching or case-insensitive searches.

javascript
const str = "Hello, JavaScript World!";

// includes — most readable
str.includes("JavaScript");  // true
str.includes("Python");      // false
str.includes("Java", 10);   // false — search from index 10

// startsWith / endsWith
str.startsWith("Hello");     // true
str.endsWith("World!");      // true

// indexOf — gives position (-1 if not found)
str.indexOf("JavaScript");   // 7
str.indexOf("Python");       // -1

// RegEx — pattern or case-insensitive
/javascript/i.test(str);     // true — case-insensitive

// Case-insensitive includes (no built-in)
str.toLowerCase().includes("javascript"); // true
Q.20 Can you modify the value of a variable captured in a Closure? ▼

Yes. A closure captures a reference to the variable, not a snapshot of its value. This means the inner function can both read and write it, and changes persist between calls. This is exactly what makes closures useful for maintaining private, stateful data.

javascript
// Closure holds a REFERENCE — changes persist across calls
function makeCounter() {
  let count = 0; // closed-over variable
  return {
    increment() { count++; },
    decrement() { count--; },
    reset()     { count = 0; },
    value()     { return count; }
  };
}

const counter = makeCounter();
counter.increment(); counter.increment(); counter.increment();
counter.value();  // 3 — count modified across multiple calls ✅
counter.decrement();
counter.value();  // 2
counter.reset();
counter.value();  // 0

// Each makeCounter() call = independent closure with own count
const c1 = makeCounter();
const c2 = makeCounter();
c1.increment(); c1.increment();
c2.increment();
c1.value(); // 2 — own count
c2.value(); // 1 — independent count ✅

Q.01 How to validate user input as they type in a form? ▼

Listen for the input event on form fields — it fires on every keystroke. Validate the current value inside the handler and give the user immediate visual feedback by updating error messages and toggling CSS classes. Use debounce for expensive validations like checking username availability via an API call.

javascript
// Real-time validation on every keystroke
const emailInput = document.querySelector("#email");
const errorMsg   = document.querySelector("#email-error");

function validateEmail(value) {
  return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(value);
}

emailInput.addEventListener("input", (e) => {
  const val   = e.target.value.trim();
  const valid = validateEmail(val);

  emailInput.classList.toggle("error", !valid && val.length > 0);
  errorMsg.textContent = (!valid && val.length > 0)
    ? "Please enter a valid email" : "";
});

// Debounced API call — check username availability
function debounce(fn, delay) {
  let timer;
  return (...args) => { clearTimeout(timer); timer = setTimeout(() => fn(...args), delay); };
}

const checkUsername = debounce(async (value) => {
  if (!value) return;
  const res  = await fetch(`/api/check-username?u=${value}`);
  const data = await res.json();
  document.querySelector("#user-status").textContent =
    data.available ? "✓ Available" : "✗ Taken";
}, 400);

document.querySelector("#username")
  .addEventListener("input", e => checkUsername(e.target.value));
Q.02 How to implement pagination for displaying large sets of data? ▼

Maintain a currentPage and itemsPerPage state. Use slice() to extract the correct chunk of data for the current page, render it, and render pagination controls. On page change, recalculate the slice and re-render. For large datasets, implement server-side pagination — pass the page number to your API and fetch only the items needed.

javascript
// Client-side pagination
const data         = [...Array(100).keys()].map(i => `Item ${i + 1}`);
const itemsPerPage = 10;
let   currentPage  = 1;

function getPage(page) {
  const start = (page - 1) * itemsPerPage;
  return data.slice(start, start + itemsPerPage);
}

function renderPage(page) {
  const items = getPage(page);
  const list  = document.querySelector("#list");
  list.innerHTML = items.map(item => `<li>${item}</li>`).join("");
  renderControls();
}

function renderControls() {
  const totalPages = Math.ceil(data.length / itemsPerPage);
  document.querySelector("#info").textContent =
    `Page ${currentPage} of ${totalPages}`;
  document.querySelector("#prev").disabled = currentPage === 1;
  document.querySelector("#next").disabled = currentPage === totalPages;
}

document.querySelector("#prev")
  .addEventListener("click", () => { currentPage--; renderPage(currentPage); });
document.querySelector("#next")
  .addEventListener("click", () => { currentPage++; renderPage(currentPage); });

renderPage(currentPage); // initial render

// Server-side pagination — fetch only needed items
async function fetchPage(page) {
  const res  = await fetch(`/api/items?page=${page}&limit=${itemsPerPage}`);
  const { items, total } = await res.json();
  renderItems(items);
}
Q.03 How to implement drag-and-drop functionality for elements on a web page? ▼

Use the native HTML Drag and Drop API. Set draggable="true" on elements, store a reference to the dragged item on dragstart, and on drop move it to the new container. Call preventDefault() in dragover to allow dropping.

javascript
// HTML: <div class="item" draggable="true">Card</div>
// HTML: <div class="dropzone">Drop here</div>

let draggedEl = null;

// On the draggable items
document.querySelectorAll(".item").forEach(item => {
  item.addEventListener("dragstart", (e) => {
    draggedEl = item;
    item.classList.add("dragging");
    e.dataTransfer.setData("text/plain", item.dataset.id);
  });
  item.addEventListener("dragend", () => {
    item.classList.remove("dragging");
    draggedEl = null;
  });
});

// On the drop zones
document.querySelectorAll(".dropzone").forEach(zone => {
  zone.addEventListener("dragover", (e) => {
    e.preventDefault(); // REQUIRED to allow drop
    zone.classList.add("drag-over");
  });
  zone.addEventListener("dragleave", () => {
    zone.classList.remove("drag-over");
  });
  zone.addEventListener("drop", (e) => {
    e.preventDefault();
    zone.classList.remove("drag-over");
    if (draggedEl) zone.appendChild(draggedEl); // move element
  });
});
Q.04 How to implement a feature that allows users to search for specific items in a large dataset? ▼

For client-side data, use Array.filter() with a case-insensitive string match on the search input value. For large server-side datasets, debounce the search input and call an API with the query, so you don't fire a request on every single keystroke.

javascript
// Client-side search — filter an array
const allProducts = [
  { id: 1, name: "Apple iPhone" },
  { id: 2, name: "Samsung Galaxy" },
  { id: 3, name: "Google Pixel" },
];

function searchProducts(query) {
  const q = query.toLowerCase().trim();
  return q
    ? allProducts.filter(p => p.name.toLowerCase().includes(q))
    : allProducts;
}

document.querySelector("#search").addEventListener("input", (e) => {
  const results = searchProducts(e.target.value);
  renderResults(results);
});

// Server-side search — debounce API call
function debounce(fn, ms) {
  let t; return (...a) => { clearTimeout(t); t = setTimeout(() => fn(...a), ms); };
}

const serverSearch = debounce(async (q) => {
  if (!q.trim()) { renderResults([]); return; }
  const res     = await fetch(`/api/products?q=${encodeURIComponent(q)}`);
  const results = await res.json();
  renderResults(results);
}, 350);

document.querySelector("#search")
  .addEventListener("input", e => serverSearch(e.target.value));
Q.05 How to implement live search suggestions as the user types? ▼

Debounce the input event, fetch matching suggestions from an API (or filter local data), then render a dropdown below the input. Hide it on blur or when the user selects a suggestion. This pattern is the core of any autocomplete or typeahead component.

javascript
// HTML: <input id="search"> <ul id="suggestions"></ul>

const input       = document.querySelector("#search");

Q.07 How to create an infinite scrolling feature when a user reaches the bottom of a webpage? ▼

The modern approach is IntersectionObserver — place a sentinel element at the bottom of your list and observe it. When it enters the viewport, load the next batch of data. This is far more efficient than listening to the scroll event, which fires hundreds of times per second.

javascript
// HTML: <div id="list"></div> <div id="sentinel"></div>

let loading  = false;
let hasMore  = true;

async function loadMore() {
  if (loading || !hasMore) return;
  loading = true;
  showSpinner();
  try {
    const res  = await fetch(`/api/posts?page=${page}&limit=10`);
    const data = await res.json();
    appendItems(data.items);
    hasMore = data.hasNextPage;
    page++;
  } finally {
    loading = false;
    hideSpinner();
  }
}

// IntersectionObserver — fires when sentinel enters viewport
const sentinel = document.querySelector("#sentinel");
const observer = new IntersectionObserver((entries) => {
  if (entries[0].isIntersecting) loadMore();
}, { threshold: 0.1 });

observer.observe(sentinel);
loadMore(); // load first batch

function appendItems(items) {
  const list = document.querySelector("#list");
  items.forEach(item => {
    const el = document.createElement("div");
    el.textContent = item.title;
    list.appendChild(el);
  });
}

Q.03 Write a function that checks whether a given string is a palindrome or not. ▼

A palindrome reads the same forwards and backwards. Reverse the string and compare it to the original. Convert to lowercase first to make the check case-insensitive. You can also use a two-pointer approach without creating a reversed copy.

  const clean    = str.toLowerCase().replace(/[^a-z0-9]/g, "");
  const reversed = clean.split("").reverse().join("");

}

// Two-pointer approach (no extra array)

  const s = str.toLowerCase();
  let left = 0, right = s.length - 1;
  while (left < right) {
    if (s[left] !== s[right]) return false;
    left++; right--;
  }
  return true;
}

Q.05 Write a function that checks whether two strings are anagrams or not. ▼

}

// Character frequency map approach (more efficient for long strings)
function isAnagramMap(str1, str2) {
  const s1 = str1.toLowerCase().replace(/\s/g, "");
  const s2 = str2.toLowerCase().replace(/\s/g, "");
  if (s1.length !== s2.length) return false;
  const map = {};
  for (const c of s1) map[c] = (map[c] || 0) + 1;
  for (const c of s2) {
    if (!map[c]) return false;
    map[c]--;
  }
  return true;
}

Q.08 Write a function to check if a given number is prime or not. ▼

A prime number is greater than 1 and divisible only by 1 and itself. The optimised approach only checks divisors up to the square root of the number, skipping even numbers after 2 — this reduces unnecessary iterations significantly.

javascript
function isPrime(n) {
  if (n < 2) return false;           // 0, 1 not prime
  if (n === 2) return true;          // 2 is prime
  if (n % 2 === 0) return false;    // even numbers > 2 not prime

  // Check odd divisors up to square root only
  for (let i = 3; i <= Math.sqrt(n); i += 2) {
    if (n % i === 0) return false;
  }
  return true;
}

isPrime(2);   // true
isPrime(7);   // true
isPrime(17);  // true
isPrime(1);   // false
isPrime(12);  // false

// Get all primes up to N (Sieve of Eratosthenes)
function getPrimesUpTo(n) {
  const sieve = new Array(n + 1).fill(true);
  sieve[0] = sieve[1] = false;
  for (let i = 2; i * i <= n; i++) {
    if (sieve[i]) {
      for (let j = i * i; j <= n; j += i) sieve[j] = false;
    }
  }
  return sieve.map((v, i) => v ? i : -1).filter(i => i > 0);
}

Q.21 Write a function to calculate the Fibonacci sequence up to a given number. ▼

The Fibonacci sequence starts with 0, 1 and each subsequent number is the sum of the two before it. Generate the sequence iteratively (efficient) or recursively (elegant but slow without memoization). Show both approaches along with the memoized recursive version.

javascript
// Iterative — returns array up to n terms
function fibonacci(n) {
  if (n <= 0) return [];
  if (n === 1) return [0];
  const seq = [0, 1];
  for (let i = 2; i < n; i++) {
    seq.push(seq[i - 1] + seq[i - 2]);
  }
  return seq;
}

// Recursive — elegant but O(2^n) without memo
function fibRecursive(n) {
  if (n <= 1) return n;
  return fibRecursive(n - 1) + fibRecursive(n - 2);
}

// Memoized recursive — O(n) time
function fibMemo(n, memo = {}) {
  if (n in memo) return memo[n];
  if (n <= 1) return n;
  memo[n] = fibMemo(n - 1, memo) + fibMemo(n - 2, memo);
  return memo[n];
}

fibonacci(8);       // [0,1,1,2,3,5,8,13]
fibRecursive(7);    // 13 (the 7th Fibonacci number)
fibMemo(50);        // 12586269025 — fast with memo ✅
